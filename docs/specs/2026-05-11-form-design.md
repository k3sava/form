# FORM — Design Spec

**Status:** Draft for review · 2026-05-11
**Owner:** Kesava · **Author:** Claude

## Goal

Turn FORM from a 9-effect poster filter into a typography lab. The user types a phrase; the system arranges it the way a typographer would — splitting on punctuation, scaling for hierarchy, choosing weight and color contrast, breaking lines with intention. Five named design philosophies (Swiss, Editorial, Brutalist, Kinetic, Painterly) act as opinionated arrangers. A Blend Lab lets the user mix philosophies on a 5-axis weight space to create new ones on demand.

Outputs: PNG still or short WebM loop. Three formats (1:1, 4:5, 1.91:1).

## Non-goals

- Not a vector poster generator (Canvas 2D, raster output).
- Not a font foundry — uses system + variable fonts available everywhere.
- Not a layout-from-prompt AI tool — pure heuristic + parser, no LLM at runtime.
- Not a "many styles" tool — exactly five philosophies, each fully realized.

## Architecture

**Multi-page static site under `github.com/k3sava/form`.** Six HTML files share a JS library:

```
form/
  index.html              ← hub (philosophy picker, live previews, blend CTA)
  swiss/index.html        ← Swiss mode of the toy
  editorial/index.html
  brutalist/index.html
  kinetic/index.html
  painterly/index.html
  blend/index.html        ← Blend Lab (5-axis sliders)
  shared/
    app.js                ← shell: toy chrome, parser glue, render loop, export
    parse.js              ← phrase parser
    philosophies/
      swiss.js
      editorial.js
      brutalist.js
      kinetic.js
      painterly.js
      blend.js
    fonts/                ← variable font subsets, if licensed
    og/                   ← per-page og.svg generated by aggregator
```

Each HTML file is a thin shell that imports `app.js` with a `data-philosophy` attribute. The shell handles topbar, controlbar, drawer, splash, theme, export — uniform across all six. The active philosophy module owns palette, type, layout, motion.

The little-toys aggregator extends `aggregate-toys.mjs` to fetch the directory listing for `form/` instead of just `index.html`, so each subpage lands at its own canonical URL on toys.iamkesava.com.

**Bundle strategy:** the philosophy modules and `compromise.js` (~150KB) load on demand. Initial HTML is splash + shell only. NLP imports kick off after the splash renders. By the time the user dismisses, parser is ready.

## Phrase parser

One pure function: `parse(text) → ParseTree`. Lives in `parse.js`. Uses [compromise.js](https://compromise.cool/) (`v14`, lazy-loaded). Returns a structured representation philosophies can interpret:

```js
{
  raw: "LESS. BETTER.",
  beats: [
    { text: "LESS",   words: ["LESS"],   tokens: [{w: "LESS", pos: "Adverb"}],   syllables: 1, emphasis: 0.5 },
    { text: "BETTER", words: ["BETTER"], tokens: [{w: "BETTER", pos: "Adjective"}], syllables: 2, emphasis: 0.9 },
  ],
  pattern: "parallel-2",     // detects same-length beats
  tone:    "imperative",     // statement|question|exclaim|imperative|fragment
  payoff:  1,                // index of beat carrying the emphasis (last is default)
  caps:    "all",            // all|title|sentence|mixed
}
```

Detection rules:
- **Beats**: split on `. , ; : ! ?` and em-dashes. Trim.
- **Parallel**: ≥2 beats, all within ±1 word count → `parallel-N`.
- **Payoff**: last beat by default; `!` on a non-last beat overrides; ALL-CAPS word in a Title-Case phrase overrides.
- **Tone**: terminal `?` → question, terminal `!` → exclaim, single-clause imperative verb → imperative, else statement/fragment.
- **Emphasis (per beat)**: 0–1. Heuristics: position weight (last gets +0.2), length weight (long words get +0.1), final-punctuation weight, ALL-CAPS-in-title overrides to 1.0.

The tree is computed once on input change and passed to whichever philosophy renders.

## Philosophy modules

Each module is a single file exporting:

```ts
export default {
  id: 'swiss',
  name: 'Swiss',
  palette: PaletteSpec,      // colors + roles (bg, fg, accent, dim)
  type:    TypeSpec,         // family list, weights, opsz, caps rules
  layout(tree, format, ctx),  // returns {boxes: Box[], grid: GridSpec}
  render(ctx, layout, t),     // paints to canvas
  motion:  MotionSpec,       // kind, intensity, looping behavior
  controls: Control[],       // sliders / pickers shown in drawer
  vector(): NumericVector,   // for blending — see Blend section
}
```

Sketched intent for each (full visual specs come in the implementation plan):

- **Swiss** — Helvetica/Akzidenz, off-white surface, primary color blocks under the payoff beat. 12-column grid; each beat anchored to a grid cell. Scale contrast from beat emphasis. Motion: tiny grid-line pulse, blocks shift one cell every few seconds. Restraint above all.
- **Editorial** — Caslon/Garamond/Sabon, classical hierarchy. Long beats become headlines; short beats become drop-caps or pull-quotes. Horizontal rules separate sections. Color: warm cream + ink + a single accent. Motion: weight axis morphs slowly between two weights of the same family.
- **Brutalist** — Helvetica Black on raw paper (`#e9e4d8` with grain), strikethroughs and misregistration as design elements, photocopier noise overlay. Text intentionally crowds edges; punctuation gets oversize treatment. Motion: scan-line glitch, occasional jam, ink density flicker.
- **Kinetic** — Variable-font play with InterVariable or Recursive. Letters individually positioned, each animating along wght/wdth/slnt axes. Neutral palette so motion carries the design. Layout follows the parse tree but is mostly typographic-grid + per-letter choreography.
- **Painterly** — Sumi ink simulation: each glyph drawn with a brush trail that bleeds into paper texture. Vertical or diagonal baselines allowed. Layout uses brushstroke curves (Bezier-driven baselines). Motion: slow ink bloom on appear, then stillness. One-shot rather than looping.

Every philosophy must clear the mycelium bar (its current quality is the visual benchmark).

## Blend Lab

**Page**: `/form/blend/`. Same chrome as the other five.

**Controls**: 5 horizontal sliders, one per philosophy, weights ∈ [0, 1]. Auto-normalized to sum to 1. A canvas-style "what's dominant" indicator shows the active blend (e.g., "Swiss 0.6 · Brutalist 0.4"). Bonus: a barycentric 5-vertex chooser for users who prefer direct geometric mixing.

**How blending works**: each philosophy returns a `vector()` of numeric parameters with stable schema:

```js
{
  // colors as Lab triples
  bg:   [L, a, b],
  fg:   [L, a, b],
  accent: [L, a, b],
  dim:  [L, a, b],
  // type
  wght: 0..1000, wdth: 50..200, opsz: 8..144, caps: 0..1,
  // layout
  gridCols: 2..16, asymmetry: 0..1, density: 0..1, hierarchy: 0..1,
  // motion
  motionKind: enum, motionAmp: 0..1, motionRate: 0..1,
  // texture
  grain: 0..1, misregister: 0..1, inkBloom: 0..1,
}
```

Continuous fields lerp across the weight vector. Discrete fields (`motionKind`, `caps` boolean) snap to the dominant philosophy (or argmax). Color triples interpolate in Lab space, not RGB, to avoid muddy blends. The shared renderer takes the blended vector and renders without knowing which philosophy provided it.

**Named blends** (optional v2): If a particular combination is interesting, it gets its own URL like `/form/swiss-painterly/` with a curated weight preset. Each adds an AEO surface.

## UI / chrome (toy-pattern match)

Every page uses the same shell:

- **Splash overlay** (`.ov` / `#splash`) — first load only. Title "FORM", tagline ("a typography lab"). 2-column grid of shortcuts (1-col on mobile). CTA row: "Enter — tap anywhere to begin". Touch-mode shortcuts auto-shown when `(hover:none) and (pointer:coarse)`. Dismisses on tap, click, or any key. Replay via `?`.
- **Top-right pill** (`#topbar`) — small `.tb` buttons:
  - `?` — open shortcuts splash (kbd: `?`)
  - `☾`/`☀` — theme toggle (kbd: `T`)
  - `✎` — open drawer (kbd: `/`)
  - `↓` — save PNG (kbd: `S`)
  - `●` — record WebM (kbd: `R`)
- **Bottom-center pill** (`#controlbar`) — `.pb` chips. On `/form/<philosophy>/` pages: 5 philosophy chips + a "BLEND" chip. Tapping a chip navigates to that philosophy's page. Active chip highlighted. On `/form/` hub: hidden. On `/form/blend/`: replaced with a slider-strip showing all 5 weights.
- **Right drawer** (`#drawer`) — slides in from right on first focus, on `/` key, on `✎` press, or on first input. Holds:
  - Text input (the phrase) — full-width, big, the centerpiece of the drawer
  - Format selector (square / portrait / landscape)
  - Per-philosophy controls (sliders/pickers defined in the philosophy module)
- **No `<header>`, no `<footer>`, no wordmark on canvas.** Matches kaleidoscopic / string-art / aurora.
- **Theme** — light/dark via `data-theme`. Persist in localStorage. Each philosophy declares whether its palette wins or whether it respects the theme. Swiss/Editorial respect; Painterly/Kinetic ride their own surface.

### Keyboard shortcuts (full set)

```
Type           edit phrase (focuses input automatically)
Enter          dismiss splash / commit current edit
Esc            close drawer / dismiss splash
Tab            next philosophy (cycles within controlbar)
Shift+Tab      prev philosophy
B              go to Blend Lab
1..5           jump to philosophy 1..5 (Swiss/Edit/Brut/Kin/Paint)
Space          toggle motion play/pause
/              focus phrase input (open drawer if closed)
T              toggle theme
S              save PNG
R              start/stop video recording
?              open shortcuts splash
```

All shortcuts visible in the splash grid. Drawer has visible kbd hints on each control.

## Accessibility

- All interactive elements have `aria-label`.
- Splash uses `role="dialog"` with `aria-modal="true"` and a focus trap (Tab cycles inside until dismissed).
- Drawer uses `role="region" aria-label="controls"`.
- Canvas has an `aria-label` that mirrors the current phrase + philosophy ("FORM poster: 'LESS. BETTER.' in Swiss style").
- Color contrast: every philosophy's text-on-bg must clear WCAG AA (≥4.5:1 for normal weight, ≥3:1 for large). Linted at build time.
- Reduced-motion: if `prefers-reduced-motion: reduce`, motion components freeze on a representative still frame.
- Keyboard-only: every action reachable without pointer.

## AEO

Six pages, six distinct landings:

| URL | Title | Intent |
|---|---|---|
| `/form/` | FORM — typography lab | hub, "generative typography tool" |
| `/form/swiss/` | Swiss poster maker — FORM | "Swiss style poster generator", "Müller-Brockmann" |
| `/form/editorial/` | Editorial poster maker — FORM | "editorial typography", "magazine poster" |
| `/form/brutalist/` | Brutalist poster maker — FORM | "zine poster generator", "brutalist type" |
| `/form/kinetic/` | Kinetic poster maker — FORM | "kinetic typography", "variable font animation" |
| `/form/painterly/` | Painterly poster maker — FORM | "sumi ink", "ink wash typography" |
| `/form/blend/` | Blend Lab — FORM | "mix typography styles", "design system blender" |

Each page gets:
- Custom `<title>`, `<meta description>`, OG image (philosophy-themed)
- `Schema.org/CreativeWork` JSON-LD identifying the philosophy
- Per-page entry in `llms.txt` and `toys.json`
- BreadcrumbList: little toys → form → philosophy
- Internal links: philosophy chips in `#controlbar` are real `<a>` tags

Aggregator changes: `aggregate-toys.mjs` adds a `subpaths` field per toy. For FORM, subpaths is `["", "swiss", "editorial", "brutalist", "kinetic", "painterly", "blend"]`. The script fetches each `<subpath>/index.html` from raw.githubusercontent.com, injects canonical + OG + JSON-LD per page, writes to `public/form/<subpath>/index.html`.

## Export

- **PNG**: `canvas.toDataURL('image/png')`. Filename `form-<philosophy>-<format>-<phrase-slug>.png`.
- **WebM**: `MediaRecorder` on canvas captureStream, 30s loop at 30fps, VP9 codec. Filename `form-<philosophy>-<format>-<phrase-slug>.webm`.
- Painterly philosophy: if motion is one-shot (ink bloom + rest), record one full cycle then loop the still.
- Pre-flight: warn user when phrase + format combination would clip text on canvas.

## Theme system

Per-philosophy palettes are defined as Lab triples in the philosophy module. The shell exposes light/dark variants. Swiss + Editorial + Kinetic respect theme; Brutalist + Painterly always use their own surface (paper texture / ink wash respectively) regardless of theme. The topbar/controlbar/drawer chrome uses the shell's neutral theme tokens (`--ov-bg`, `--ov-text`, etc.) regardless of philosophy.

## Phasing (what ships first)

Cuts honest, not roadmap:

1. **Phase 0 — chrome rewrite.** Replace current header/sidebar/footer with topbar+controlbar+drawer+splash. Keep current effects under a single "DRAFT" philosophy until the rest are built. Ship as `/form/`. Verifies the new chrome works.
2. **Phase 1 — Swiss + Editorial.** Two philosophies fully fleshed out. Phrase parser online. Each at its own URL. Drawer holds per-philosophy controls. Mycelium-quality bar enforced.
3. **Phase 2 — Brutalist + Kinetic.** Two more philosophies. Variable font infrastructure introduced for Kinetic.
4. **Phase 3 — Painterly.** Ink simulation (the most R&D-heavy). Ships only when it clears the bar.
5. **Phase 4 — Blend Lab.** Blend math, sliders, named blends.
6. **Phase 5 — AEO polish.** Per-page metadata, og.svg per philosophy, sitemap/llms.txt entries.

Each phase ships independently. We can pause after any phase and the toy remains useful.

## Open items / risks

- **Variable font licensing**: Phase 2 Kinetic needs a variable font with the right axes available for embed. Recursive is OFL — works. InterVariable likewise. Need to confirm Casual axis exists for Editorial morph. (Resolvable.)
- **Sumi ink simulation**: Phase 3 quality is uncertain. May need to ship Painterly as a "still-only with on-appear bloom" rather than a continuously-flowing motion.
- **NLP bundle**: 150KB of compromise.js is large for a static toy. Lazy-load mitigates. If it still feels heavy on slow connections, a smaller heuristic-only path (no compromise) can ship as fallback.
- **Aggregator extension**: pulling subpaths from a GitHub repo means listing directories — needs a small change to `aggregate-toys.mjs` to either hard-code the FORM subpath set or use the GitHub contents API to list.

## What we are NOT doing in this spec

Out of scope for this rewrite, on the wishlist for the private explorer (option C from brainstorming):

- Reading a blog URL and extracting the phrase automatically
- Generating multiple candidate compositions and picking one
- LLM-driven layout suggestions
- User accounts, saved compositions, libraries

Those happen in the C explorer once the toy is solid.
